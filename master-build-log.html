Build standards
Conventions applied across all portfolio tools
StandardRule
3-piece structureEvery tool ships with: Launch the tool · Theory companion · Build journal. This 3-piece structure is the standard for all portfolio items going forward.
Zero-server architectureAll tools run browser-native. No server, no database, no authentication. Data never leaves the machine. Non-negotiable.
Single HTML or folderTools are either single self-contained HTML files (bundled) or a folder of HTML + JS files (for GitHub Pages). No build pipelines, no npm, no frameworks.
CDN-only dependenciesAll libraries loaded from cdnjs.cloudflare.com or cdn.jsdelivr.net. Version-pinned. No package.json.
Build journal formatNarrative-first, PM lifecycle structure, no textbook references in public-facing content, grounded in lived consulting experience. DM Serif Display / DM Sans / JetBrains Mono typefaces, dark header, warm paper background.
Theory companion formatComprehensive theoretical framework for the tool's analytical domain. References source material explicitly. Same typeface stack as build journal. Self-contained HTML, sidebar navigation, scroll tracking.
Test coverageEvery module has a test suite validating core engine logic against known test cases. Analytical formulas validated to 4 decimal places where applicable.
FontsPortfolio landing page and build documents: DM Serif Display + DM Sans + JetBrains Mono. Tool UIs: Syne + Instrument Sans + JetBrains Mono.
